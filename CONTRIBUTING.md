# 贡献指南

感谢你对本项目的关注！以下是参与贡献的方式。

## 如何贡献

### 报告 Bug

使用 [Bug Report Template](https://github.com/HEXUXIU/M365-Copilot2API/issues/new?template=bug_report.yml) 提交问题。

### 建议功能

使用 [Feature Request Template](https://github.com/HEXUXIU/M365-Copilot2API/issues/new?template=feature_request.yml) 提交想法。

### 提交代码

1. Fork 仓库
2. 创建特性分支: `git checkout -b feature/your-feature`
3. 提交改动: `git commit -m "feat: add your feature"`
4. 推送分支: `git push origin feature/your-feature`
5. 创建 Pull Request

## 代码风格

- 遵循 [PEP 8](https://peps.python.org/pep-0008/)
- 函数和变量使用 `snake_case`
- 类使用 `PascalCase`
- 常量使用 `UPPER_CASE`
- 不使用类型注解（保持 Python 3.10+ 兼容性）
- 除非必要，不添加新的依赖

## Commit 规范

- `feat:` 新功能
- `fix:` 修复 Bug
- `docs:` 文档更新
- `refactor:` 代码重构
- `chore:` 杂项

## 行为准则

请阅读我们的 [Code of Conduct](CODE_OF_CONDUCT.md)。

## 重要提醒

- 本项目仅用于学习和研究，请勿提交商业用途代码
- 不要提交任何个人凭证、token 或 cookie
- 不要提交针对特定用户数据的代码
